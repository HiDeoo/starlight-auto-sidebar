The “Demo” group visible in the sidebar is an [autogenerated group](https://starlight.astro.build/guides/sidebar/#autogenerated-groups) defined using the following configuration:

```js
starlight({
  sidebar: [
    {
      label: 'Demo',
      autogenerate: { directory: 'demo' },
    },
  ],
})
```

The `src/content/docs/demo/` directory contains the following files and directories:

import { FileTree } from '@astrojs/starlight/components'

<FileTree>

- docs
  - …
  - demo
    - api
      - order.md
      - product.md
      - user.md
    - changelog
      - v1-0-0.md
      - v2-0-0.md
      - v3-0-0.md
    - overview.mdx
  - …

</FileTree>

By default, the following sidebar group will be generated by Starlight:

import SidebarPreview from '../../../components/SidebarPreview.astro'

<SidebarPreview
  config={[
    {
      label: 'Demo',
      items: [
        { label: 'Overview' },
        { label: 'api', items: [{ label: 'Order' }, { label: 'Product' }, { label: 'User' }] },
        { label: 'changelog', items: [{ label: 'v1.0.0' }, { label: 'v2.0.0' }, { label: 'v3.0.0' }] },
      ],
    },
  ]}
/>

The Starlight Auto Sidebar plugin is used to customize various aspects of autogenerated sidebar groups:

1. [Rename the `changelog` directory sidebar label to `Release Notes`](#rename-the-changelog-directory)
1. [Change the sorting order of the `changelog` directory to be reversed](#change-the-sorting-order-of-the-changelog-directory)
1. [Rename the `api` directory sidebar label to `API`](#rename-the-api-directory)
1. [Add an `Experimental` caution badge to the `api` directory sidebar label](#add-an-experimental-caution-badge-to-the-api-directory)
1. [Collapse the `api` directory in the sidebar](#collapse-the-api-directory)

## Steps

Here are all the steps to customize the autogenerated sidebar groups to match the rendered sidebar:

import { Steps } from '@astrojs/starlight/components'

<Steps>

1. #### Rename the `changelog` directory

   To rename the `changelog` directory sidebar label to `Release Notes`, a [metadata file](/metadata/) located in the `src/content/docs/reference/changelog/` directory is used.

   The [`label`](/metadata/#label) property is used to set the new label:

   ```yml title="src/content/docs/reference/changelog/_meta.yml"
   # Set the directory sidebar label to `Release Notes`.
   label: Release Notes
   ```

1. #### Change the sorting order of the `changelog` directory

   By default, content in in autogenerated sidebar groups is sorted by user-specific [order](https://starlight.astro.build/reference/frontmatter/#order) defined in a frontmatter field or alphabetically by slug.
   For a directory containing release notes, it might be useful to reverse the sorting order so that the latest release notes are displayed first.

   To change the sorting order of the `changelog` directory, a [metadata file](/metadata/) located in the `src/content/docs/reference/changelog/` directory is used.
   The [`sort`](/metadata/#sort) property is used to control the sorting order of the content in the directory:

   ```yml title="src/content/docs/reference/changelog/_meta.yml"
   # Reverse the sorting order of the directory content.
   sort: reverse-slug
   ```

1. #### Rename the `api` directory

   To rename the `api` directory sidebar label to `API`, a [metadata file](/metadata/) located in the `src/content/docs/reference/api/` directory is used.

   The [`label`](/metadata/#label) property is used to set the new label:

   ```yml title="src/content/docs/reference/api/_meta.yml"
   # Set the directory sidebar label to `API`.
   label: API
   ```

1. #### Add an `Experimental` caution badge to the `api` directory

   To add an `Experimental` caution badge to the `api` directory sidebar label, a [metadata file](/metadata/) located in the `src/content/docs/reference/api/` directory is used.

   The [`badge`](/metadata/#badge) property is used to add a badge to the directory sidebar label:

   ```yml title="src/content/docs/reference/api/_meta.yml"
   # Add a caution badge to the directory sidebar label.
   badge:
     text: Experimental
     variant: caution
   ```

1. #### Collapse the `api` directory

   To collapse the `api` directory in the sidebar, a [metadata file](/metadata/) located in the `src/content/docs/reference/api/` directory is used.

   The [`collapsed`](/metadata/#collapsed) property is used to make the directory collapsed in the sidebar:

   ```yml title="src/content/docs/reference/api/_meta.yml"
   # Collapse the directory in the sidebar.
   collapsed: true
   ```

</Steps>

Check out the [metadata reference](/metadata/) to learn more about all the available customization options.
