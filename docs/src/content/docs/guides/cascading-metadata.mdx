---
title: Cascading Metadata
description: Learn how to cascade metadata properties to nested directories in autogenerated sidebar groups.
---

By default, all properties in a metadata file apply only to the directory containing the metadata file.
Cascading metadata allows you to apply some properties to nested directories when displayed in an autogenerated sidebar group.

## Cascade metadata

To cascade some properties to nested directories, use the [`cascade`](/metadata/#cascade) property in the metadata file and specify the properties you want to apply to all nested directories.

For example, given the following Starlight `src/content/docs/` directory:

import { FileTree } from '@astrojs/starlight/components'

<FileTree>

- docs
  - guides
    - example.md
  - reference
    - about.md
    - api-reference
      - api.md
      - admin
        - api.md
    - changelog
      - 2025.md
      - 2024.md

</FileTree>

And the given [autogenerated sidebar group](https://starlight.astro.build/guides/sidebar/#autogenerated-groups) configuration for the `reference/` directory:

```js
starlight({
  sidebar: [
    {
      label: 'Reference',
      autogenerate: { directory: 'reference' },
    },
  ],
})
```

By default, the following sidebar group will be generated:

import SidebarPreview from '../../../components/SidebarPreview.astro'

<SidebarPreview
  config={[
    {
      label: 'Reference',
      items: [
        { label: 'About' },
        { label: 'api-reference', items: [{ label: 'API' }, { label: 'admin', items: [{ label: 'API' }] }] },
        { label: 'changelog', items: [{ label: '2024' }, { label: '2025' }] },
      ],
    },
  ]}
/>

To collapse the `api-reference` directory, create a metadata file named `_meta.yml` in the `src/content/docs/reference/api-reference/` directory with the following content:

```yml title="src/content/docs/reference/api-reference/_meta.yml"
# Collapse the directory in the sidebar.
collapsed: true
```

The updated sidebar will now look like this:

<SidebarPreview
  config={[
    {
      label: 'Reference',
      items: [
        { label: 'About' },
        {
          label: 'api-reference',
          items: [{ label: 'API' }, { label: 'admin', items: [{ label: 'API' }] }],
          collapsed: true,
        },
        { label: 'changelog', items: [{ label: '2024' }, { label: '2025' }] },
      ],
    },
  ]}
/>

If you expand the `api-reference` directory, you can notice that the `admin` directory is not collapsed.
This is expected as properties in a metadata file apply only to the directory containing the metadata file by default.

To also collapse the `admin` directory, you can cascade the `collapsed` property to all nested directories using the `cascade` property in the same metadata file:

```yml title="src/content/docs/reference/api-reference/_meta.yml"
# Collapse the directory in the sidebar.
collapsed: true
# Cascade the `collapsed` property to all nested directories.
cascade:
  - collapsed
```

The updated sidebar will now look like this:

<SidebarPreview
  config={[
    {
      label: 'Reference',
      items: [
        { label: 'About' },
        {
          label: 'api-reference',
          items: [{ label: 'API' }, { label: 'admin', items: [{ label: 'API' }], collapsed: true }],
          collapsed: true,
        },
        { label: 'changelog', items: [{ label: '2024' }, { label: '2025' }] },
      ],
    },
  ]}
/>

When expanding the `api-reference` directory, you can see that the `admin` directory is now collapsed as well.

To learn more about all the available properties that can be cascaded to nested directories, check the [metadata reference](/metadata/#cascade).
