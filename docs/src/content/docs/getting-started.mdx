A [Starlight](https://starlight.astro.build) plugin to tweak [autogenerated sidebar groups](https://starlight.astro.build/guides/sidebar/#autogenerated-groups):

- [Rename](/metadata/#label) autogenerated sidebar group directories.
- Add [badges](/metadata/#badge) or change the [collapsed](/metadata/#collapsed) state of autogenerated sidebar group directories.
- Manually specify the [order](/metadata/#order) of autogenerated sidebar group directories.
- Change the [sorting](/metadata/#sort) behavior of autogenerated sidebar groups.
- [Hide](/metadata/#hidden) autogenerated sidebar group directories.
- Limit the [depth](/metadata/#depth) of autogenerated sidebar groups.
- [Cascade](/metadata/#cascade) some options to all nested autogenerated sidebar group directories.

Check out the [demo](/demo/overview/) for a preview of various features provided by the plugin.

## Prerequisites

You will need to have a Starlight website set up.
If you don't have one yet, you can follow the ["Getting Started"](https://starlight.astro.build/getting-started/) guide in the Starlight docs to create one.

## Installation

import { Steps } from '@astrojs/starlight/components'
import { PackageManagers } from '@hideoo/starlight-plugins-docs-components'
import { FileTree } from '@astrojs/starlight/components'
import SidebarPreview from '../../components/SidebarPreview.astro'

<Steps>

1. Starlight Auto Sidebar is a Starlight [plugin](https://starlight.astro.build/reference/plugins/). Install it using your favorite package manager:

   <PackageManagers pkg="starlight-auto-sidebar" />
   <br />

2. Add the plugin to your Starlight [configuration](https://starlight.astro.build/reference/configuration/#plugins) in the `astro.config.mjs` file.

   ```js title="astro.config.mjs" ins={4,9}
   // @ts-check
   import starlight from '@astrojs/starlight'
   import { defineConfig } from 'astro/config'
   import starlightAutoSidebar from 'starlight-auto-sidebar'

   export default defineConfig({
     integrations: [
       starlight({
         plugins: [starlightAutoSidebar()],
         title: 'My Docs',
       }),
     ],
   })
   ```

3. Starlight Auto Sidebar uses Astroâ€™s [content collections](https://docs.astro.build/en/guides/content-collections/), which are configured in the `src/content.config.ts` file.

   Update the content config file to add support for customizing autogenerated sidebar groups using metadata files:

   ```js title="src/content.config.ts" ins={4,5,9-12}
   import { docsLoader } from '@astrojs/starlight/loaders'
   import { docsSchema } from '@astrojs/starlight/schema'
   import { defineCollection } from 'astro:content'
   import { autoSidebarLoader } from 'starlight-auto-sidebar/loader'
   import { autoSidebarSchema } from 'starlight-auto-sidebar/schema'

   export const collections = {
     docs: defineCollection({ loader: docsLoader(), schema: docsSchema() }),
     autoSidebar: defineCollection({
       loader: autoSidebarLoader(),
       schema: autoSidebarSchema(),
     }),
   }
   ```

4. Starlight Auto Sidebar uses metadata files located in the directory of an autogenerated sidebar group that you want to customize.

   For example, given the following Starlight `src/content/docs/` directory:

   <FileTree>

   - docs
     - guides
       - example.md
     - reference
       - about.md
       - api-reference
         - api.md

   </FileTree>

   And the given [autogenerated sidebar group](https://starlight.astro.build/guides/sidebar/#autogenerated-groups) configuration for the `reference/` directory:

   ```js
   starlight({
     sidebar: [
       {
         label: 'Reference',
         autogenerate: { directory: 'reference' },
       },
     ],
   })
   ```

   By default, the following sidebar group will be generated:

   <SidebarPreview
     config={[
       {
         label: 'Reference',
         items: [{ label: 'About' }, { label: 'api-reference', items: [{ label: 'API' }] }],
       },
     ]}
   />

   To update the `api-reference` directory label to `API Reference`, create a metadata file named `_meta.yml` in the `src/content/docs/reference/api-reference/` directory with the following content:

   ```yml title="src/content/docs/reference/api-reference/_meta.yml"
   # Set the directory sidebar label to `API Reference`.
   label: API Reference
   ```

5. [Start the development server](https://starlight.astro.build/getting-started/#start-the-development-server) to preview the updated sidebar which should now look like this:

   <SidebarPreview
     config={[
       {
         label: 'Reference',
         items: [{ label: 'About' }, { label: 'API Reference', items: [{ label: 'API' }] }],
       },
     ]}
   />

</Steps>

To learn more about metadata files and how to customize autogenerated sidebar groups, check the [metadata reference](/metadata/).
